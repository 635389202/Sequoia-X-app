# Sequoia-X Android App Design

## Goal

Build a native Android app for reading Sequoia-X stock selection results on mobile. The first version is a fast local reader: it imports the existing `sequoia_app_data_*.zip` package, stores the data on device, and lets the user browse, filter, sort, and inspect selected stocks without relying on a live web page.

The app is not a trading terminal and does not place orders. It is an offline-first watchlist and research viewer for strategy results generated by the desktop pipeline.

## First-Version Scope

The first version must support:

- Importing the existing exported zip package from local storage.
- Reading `manifest.json`, `results.jsonl`, `stock_basic.jsonl`, `stock_daily.jsonl`, `stock_context.jsonl`, and `strategy_notes.json`.
- Showing the latest selected-stock list with stock code, name, strategy, latest price, 5/20/60-day changes, sector, and recent major information.
- Searching by code, name, sector, or major information.
- Filtering by strategy and trend preset.
- Sorting by strategy, price high to low, price low to high, and 5/20/60-day changes.
- Opening a stock detail screen with recent price trend, strategy hits, sector, major information, and price summary.
- Showing the current imported data date and import history.

The first version will not include user accounts, brokerage integration, cloud sync, automatic push notifications, or real-time quotes.

## Platform And Stack

- Android native app.
- Kotlin.
- Jetpack Compose for UI.
- Room over SQLite for local storage.
- Kotlin coroutines for import and database work.
- Android Storage Access Framework for choosing the zip file.

This stack keeps the app maintainable, gives good mobile performance, and leaves a clean path for later background updates.

## Data Model

Room stores imported data in normalized tables:

- `ImportBatch`: package id, generated time, latest date, source filename, imported time.
- `StockBasic`: symbol, name, exchange, status, type, updated time.
- `StockDaily`: symbol, date, open, high, low, close, volume, turnover.
- `StrategyResult`: result date, strategy, symbol.
- `StockContext`: symbol, sector, major information, updated time.
- `StrategyNote`: strategy, label, explanation, advice.

Indexes:

- `StockDaily(symbol, date)`.
- `StrategyResult(date, strategy, symbol)`.
- `StrategyResult(symbol)`.
- `StockBasic(symbol)`.
- `StockContext(symbol)`.

The import process should use transactions so a failed import does not leave a half-written data set.

## Performance Requirements

Performance is a first-version requirement.

- The home screen should show the latest list in about 1 second after app launch on a normal Android phone.
- Filtering and sorting should feel immediate for the current result set.
- The list must use Compose `LazyColumn` and avoid rendering off-screen rows.
- The home list must not load all historical `stock_daily` rows into memory.
- SQL should provide the latest result list, search, filter, and sort where practical.
- Each list row should only load the latest price, precomputed change fields, sector, and a short major-information preview.
- Recent sparkline data should be limited to the latest 30 closes for list rows.
- Detail screens may query longer history on demand.
- Zip parsing and database writes must run on background dispatchers with visible import progress.
- Long major-information text is truncated on list rows and shown fully on detail screens.

To support fast listing, the app may store an import-derived `ResultDisplayRow` or equivalent cache table containing latest close, 5/20/60-day changes, and recent 30-day close values for selected symbols. This avoids repeated expensive aggregation while scrolling.

## Screens

### Home

Home is the primary screen and opens directly to the latest imported result set.

Visible controls:

- Search box.
- Strategy filter.
- Trend filter.
- Sort selector.
- Import status/date indicator.

Each row shows:

- Symbol and name.
- Strategy label.
- Latest price.
- 5/20/60-day changes.
- Sector.
- Recent major information preview.
- A compact recent trend visual.

### Detail

The detail screen shows:

- Symbol and name.
- Strategy hits across imported result dates.
- Latest price summary.
- Recent price chart.
- Sector.
- Full recent major information.
- Strategy explanation and advice.

### Data

The data screen shows:

- Current latest date.
- Imported package metadata.
- Import history.
- Button to import a new zip package.

Import errors should be clear, for example: invalid zip, missing manifest, incompatible format version, or database write failure.

## Import Flow

1. User taps import.
2. Android file picker opens.
3. App validates the zip package.
4. App reads the manifest and checks `format_version`.
5. App parses JSONL files in streaming mode where possible.
6. App writes normalized rows inside a database transaction.
7. App builds or refreshes display rows for the latest result date.
8. App reports success and navigates to the latest results.

If import fails, the previous successful data remains available.

## Update Path After First Version

The next update step can add a configurable data source:

- Download zip from a fixed URL.
- Import from a synced folder.
- Import from cloud drive provider selected by Android file picker.

Background daily update and notifications should come after the manual import app is stable.

## Testing

Unit tests:

- Zip manifest validation.
- JSONL parsing.
- Price-change calculation.
- Display-row generation.
- Sort and filter query behavior.

Instrumentation or Robolectric tests:

- Import a small sample zip.
- Verify home list renders expected rows.
- Verify sort high-to-low and low-to-high.
- Verify detail screen loads without fetching all historical rows.

Manual smoke test:

- Install app on Android emulator or phone.
- Import `exports/app/sequoia_app_data_2026-07-01.zip`.
- Confirm `600601 方正科技` displays latest close `13.97` for `2026-07-01`.
- Confirm price sort high-to-low starts with the highest-priced result and low-to-high starts with the lowest-priced result.

## Acceptance Criteria

- A debug APK can be built and installed.
- The app can import the current exported zip package.
- Home list displays latest results with sector and major information.
- Search, filters, and sorting work on the latest result set.
- Stock detail opens from a row and shows recent history.
- Import does not block the UI thread.
- A failed import leaves the previous data intact.
- Automated tests cover import parsing, display row generation, and sorting behavior.
